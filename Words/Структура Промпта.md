ТВОЯ ЗАДАЧА:
Преобразовать список слов в готовый TypeScript код вызова функции `createRichWord`.

ФОРМАТ ID (ВАЖНО):
Для поля id используй строго формат: `уровень_слово` в нижнем регистре.
Примеры:
- Word: "Apple", Level: A1 -> id: 'a1_apple'
- Word: "Give up", Level: B2 -> id: 'b2_give_up'
- Word: "Scrutinize", Level: C1 -> id: 'c1_scrutinize'

АЛГОРИТМ:
1. Проанализируй входной список слов.
2. ОПРЕДЕЛЕНИЕ УРОВНЯ CEFR (КРИТИЧЕСКИ ВАЖНО):
Проблема: ИИ часто завышает уровни. Не делай этого.
Правило занижения: Если ты сомневаешься между B2 и C1, ставь B2. Если между B1 и B2, ставь B1.
Критерии:
A1-A2 (Survival): Конкретные предметы, базовые действия, семья, еда, цвета. То, что учит школьник в 1-3 классе.
B1 (Threshold): Абстрактные, но частотные понятия (Attitude, Development, Suggest). Слова, необходимые для жизни и работы.
B2 (Vantage): Более сложные, но общеупотребительные слова (Significant, Potential, Persuade). Язык новостей и YouTube.
C1 (Advanced): Слова с оттенками смысла, которые можно заменить на простые синонимы (Scrutinize -> Examine, Ubiquitous -> Everywhere).
C2 (Mastery): Книжные, устаревшие, узкоспециализированные или крайне редкие слова (Ephemeral, Cacophony, Ennui).
Распределение: В обычном тексте слов C1/C2 должно быть меньшинство (<15%). Основная масса — это B1 и B2.
3. Сгруппируй итоговый код по уровням (Сначала блок // --- LEVEL A1 ---, затем A2...).
4. Сгенерируй код.

СТРОГИЕ ТРЕБОВАНИЯ К ПОЛЯМ (TypeScript Errors Prevention):
1. Frequency может быть ТОЛЬКО: 'High', 'Medium', 'Low'.
2. Register может быть ТОЛЬКО: 'Neutral', 'Formal', 'Informal', 'Slang', 'Literary'.
3. PartOfSpeech (POS) пиши полностью: 'noun', 'verb', 'adj', 'adv', 'phrasal verb'.
4. В текстах примеров экранируй одиночные кавычки обратным слешем (Don\'t), так как код обернут в одиночные кавычки.

ТРЕБОВАНИЯ К КОНТЕНТУ:
1. Transcription: Британская IPA.
3. RussianTransliteration: Транслитерация кириллицей (как это читается по-русски), например "эмбигьюэс".
3. Translation: Основной, самый частотный перевод на русский.
4. Definition: Полное, понятное определение слова на русском языке (10-20 слов). Что оно значит и т.д. Если у слов несколько значений, то перенеси другое значение на следующую строку \n. (10-20 слов).
5. Examples: Строго 4 примера.
   - Уровень A1-A2: Простые предложения, но не примитивные (8+ слов).
   - Уровень B1-C2: Сложные, богатые контекстом предложения (10-15 слов).
   - Перевод примеров должен быть естественным русским языком.
6. UsageContext (ВАЖНО): 3 короткие строки, разделенные спецсимволом \n.
Формат: 
  - Категория (на русском): "English Phrase" (Русский перевод)'.
  - Ты обязан перевести и категорию контекста, и саму фразу.
  - Пример правильного вывода: 
    Тема: "Talk about films" (Говорить о фильмах)\n- Приблизительность: "About 5 o\'clock" (Около 5 часов)\n- Местоположение: "Walk about the town" (Гулять по городу)'
7. SecondaryMeanings (ОБЯЗАТЕЛЬНОЕ ПОЛЕ):
   - Большинство английских слов многозначны. Ты ОБЯЗАН найти до 3-х дополнительных значений.
   - Исключение: Только узкоспециализированные термины (например, "isotope" или "photosynthesis"), у которых физически нет второго смысла.
   - Для обычных слов (Run, Table, Fire, Get, Set и т.д.) отсутствие этого блока считается ОШИБКОЙ.
   - Формат: { meaning: "Краткое описание смысла на английском", translation: "Перевод этого смысла", example: { en: "Пример", ru: "Перевод" } }

ШАБЛОН ВЫВОДА:
